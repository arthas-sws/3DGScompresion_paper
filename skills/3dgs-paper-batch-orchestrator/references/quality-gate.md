# 单篇报告质量门禁

## 1. 语言门禁

默认要求：

- 中文字符不少于 500；
- 中文字符占中文与拉丁字母总和不低于 0.25；
- 不出现连续 3 个以上英文主章节标题；
- 不出现大段未解释英文正文；
- 最终结论必须是中文。

技术名词、方法名、数据集名、代码和引用不计为语言错误。

## 2. 结构门禁

必须检测：汇报摘要、核心贡献、方法、实验设置、主要结果、结果解读、局限、代码或复现状态、最终总结。

默认精读还应检测：效率与存储、消融、结果证明了什么、结果尚未证明什么。

## 3. 结果门禁

有论文结果时必须包括：数据集、指标、本文结果、baseline、可比性、证据位置和中文解释。

论文没有提供时，必须明确写“论文未报告”“无法核实”或“现有材料不足”，不能无声省略。

## 4. 证据门禁

至少存在一种：

- `[论文 ...]`
- `[补充材料 ...]`
- `[代码 ...]`
- Table / Fig. / Eq. 定位
- “论文未报告”“当前无法核实”

出现 PSNR、SSIM、LPIPS、FPS、MB、GB、ms、ATE、RPE 等指标或单位，但完全没有证据位置时，判为失败。

## 5. 独立性门禁

检查标题和正文方法是否一致，是否出现其他论文的大段残留，数字是否属于当前论文。

## 6. 建议最低中文字符

| 模式 | 最低值 |
|---|---:|
| 中文精读 | 1200 |
| 审稿 | 1200 |
| 复现 | 1000 |
| 精炼速读 | 500 |
| 元信息 | 200 |

长度只是警报，不代替内容检查。

## 7. 结果等级

- `PASS`：可进入汇总；
- `WARN`：可进入，但汇总中标注限制；
- `FAIL`：不得进入正式比较矩阵；
- `FATAL`：论文身份或来源错误，或内容严重串稿。
