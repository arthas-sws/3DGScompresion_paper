# 批量调度协议

## 1. 两阶段架构

### 阶段 A：单篇分析

每篇论文单独完成：全文获取、重新读取单篇 Skill、分析、保存、校验、必要时重试、锁定通过版。

### 阶段 B：批次汇总

只读取通过质量门禁的单篇报告，生成批次索引、分类、创新矩阵、结果矩阵、复现状态、共性问题和选型结论。

阶段 A 完成前，不得开始正式阶段 B。

## 2. 任务隔离

一个任务包只能包含：

```yaml
paper_id:
title:
source:
code:
mode:
language: zh-CN
focus:
output_path:
```

禁止包含其他论文全文、未核实数字、其他报告全文和未验证的跨论文排名。

## 3. 分片

| 模式 | 每个子批次 |
|---|---:|
| 中文精读 | 3—8 篇 |
| 审稿 | 3—6 篇 |
| 复现调研 | 2—5 篇 |
| 精炼速读 | 5—15 篇 |
| 元信息 | 10—30 篇 |

父批次汇总仅在子批次都完成后进行。

## 4. 重试

重试流程：读取失败报告和质量 JSON，保留正确内容，修复指定问题，重新完整输出，再次校验。

## 5. 文件不可覆盖

```text
P001.attempt-1.md
P001.attempt-2.md
P001.md
```

`P001.md` 只代表通过质量门禁的最终版本。

## 6. 汇总来源

正式汇总只能读取：

- `items/<id>.md`
- `validation/<id>.json`
- `manifest.json`

不得从未通过的 attempt 中提取数字。
