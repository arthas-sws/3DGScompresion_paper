# Improvement Rubric

Improvement ideas must be concrete, tied to innovation claims, and testable.

Each idea must include:

- `idea_id`;
- `idea`;
- `targets_claim`;
- `research_value`;
- `implementation_cost`;
- `failure_risk`;
- `required_resources`;
- `minimum_validation`;
- `priority`;
- `evidence_ids`.

Priority rules:

- `P0`: high value with low or medium cost and a quick validation path.
- `P1`: high value but higher cost or risk.
- `P2`: exploratory long-term direction.

Each proposed experiment must include:

- `experiment_id`;
- `claim_id`;
- `goal`;
- `hypothesis`;
- `independent_variables`;
- `control_baselines`;
- `datasets`;
- `metrics`;
- `implementation_steps`;
- `expected_result`;
- `failure_interpretation`;
- `estimated_cost`;
- `priority`.

Avoid generic suggestions like "add more ablations" unless the missing module, baseline, metric, or dataset is named.
