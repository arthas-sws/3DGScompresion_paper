# 中文输出模式

## 1. 中文精读报告（默认）

适用于“分析、总结、精读”。

必须包含：

1. 汇报摘要；
2. 论文信息与分析范围；
3. 一句话贡献；
4. 问题与动机；
5. 方法流程；
6. 技术细节；
7. 论文—代码映射；
8. 实验设置；
9. 主要结果总表；
10. 效率与存储结果；
11. 逐场景一致性；
12. 消融；
13. 定性结果和失败案例；
14. 结果证明了什么；
15. 结果尚未证明什么；
16. 结果汇报总结；
17. 相关工作；
18. 局限性；
19. 可复现性；
20. 最终汇报总结。

不设置固定字数。内容长度由论文复杂度决定。

## 2. 精炼速读

适用于“简单说说、快速了解”。

目标约 800—1800 个中文字符，但准确性优先。

必须包含：

### 核心速览

| 项目 | 结论 |
|---|---|
| 解决问题 | |
| 核心方法 | |
| 最重要结果 | |
| 主要代价 | |
| 适用场景 | |

### 关键创新

3—5 条。

### 主要结果

最多 5—8 行结果表。

### 结果解读

必须说明：

- 提升多少；
- 是否稳定；
- 代价是什么；
- 结果是否支撑主张。

### 局限和代码状态

### 一句话结论

不能因为是“精炼”模式而删除结果汇报。

## 3. 学术长文 / HTML

适用于组会、博客、深度解读。

结构：

1. 汇报摘要；
2. 元信息与来源；
3. 背景和基线谱系；
4. 预备知识；
5. 完整方法；
6. 关键公式；
7. 论文—代码映射；
8. 实验设置；
9. 定量结果；
10. 效率与存储；
11. 消融与定性结果；
12. 结果解释；
13. 讨论；
14. 局限性；
15. 可复现性；
16. 最终汇报总结。

先生成 Markdown，再转换 HTML：

```bash
python scripts/render_html.py report.md report.html \
  --template assets/article-template.html
```

HTML 只是交付形式，Markdown 是权威内容源。

## 5. 复现模式

适用于安装、运行、复现实验。

必须包含：

1. 论文和代码精确版本；
2. 仓库状态；
3. 环境；
4. 数据集目录；
5. 预处理；
6. 训练命令；
7. 评估命令；
8. 配置表；
9. 论文—代码映射；
10. 论文报告的目标结果；
11. 最低可接受复现结果；
12. 日志和输出文件；
13. sanity check；
14. 常见错误；
15. 用户硬件适配；
16. 最小复现方案；
17. 完整复现方案；
18. 复现结果汇报模板。

区分：

- 仓库原始命令；
- 根据代码构造的命令；
- 用户环境需要修改的命令。

## 6. 多论文对比

所有论文使用相同字段。

### 方法对比

| 维度 | 方法 A | 方法 B | 方法 C |
|---|---|---|---|
| 任务 | | | |
| Primitive | | | |
| Renderer | | | |
| Density control | | | |
| Loss | | | |
| Training cost | | | |
| Rendering cost | | | |
| Storage | | | |
| Code | | | |
| Limitations | | | |

### 结果对比

| 数据集/设置 | 指标 | A | B | C | 可比性 | 结论 |
|---|---|---:|---:|---:|---|---|

### 综合结果总结

必须回答：

- 哪个方法质量最好；
- 哪个方法速度最好；
- 哪个方法存储最小；
- 哪些数字不可直接比较；
- 不同应用场景该选哪个；
- 当前证据不足以判断什么。

跨论文数字必须标注“非受控对比”，除非来自统一复现实验。

### 场景选择审计

- 完整公开场景；
- 论文选用场景；
- 论文未选场景；
- 作者是否说明规则；
- 选择是否与论文分类相关；
- 可能造成的偏差；
- 不推断作者主观意图。

### 主张—证据矩阵

| 主张 | 论文证据 | 代码证据 | 可信度 | 缺失验证 |
|---|---|---|---|---|

### Rebuttal 清单

将每个主要问题转换为：

- 需要作者澄清什么；
- 需要补什么实验；
- 什么证据可以解决；
- 不解决会影响什么结论。
